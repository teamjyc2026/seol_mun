# 튜터링 멀티에이전트 v2 PRD

## 1. 문제 정의

v1 에이전트는 "물어보면 답하는" 단발 Q&A였다. 학습은 루프다:
**출제 → 학생이 답함 → 정오 판정 + 이유 설명 → 비슷한 문제로 반복**.
또한 내신 대비의 핵심인 **본문 암기 확인**과 **독해력 점검**을 담당하는 에이전트가 없었다.

v1의 기술적 결함 두 가지가 루프를 불가능하게 했다:
1. 에이전트가 **대화 히스토리를 못 봄** — 학생이 "③번"이라 답해도 어떤 문제의 답인지 모름.
2. 라우팅이 키워드 단발 — 답변 턴("③번이요")이 직전 에이전트로 이어지지 않음.

## 2. 아키텍처 (supervisor-router 멀티에이전트)

```
사용자 메시지
   ↓
[Supervisor] classifyAgent — ① 키워드 룰(LLM 0회) ② sticky(직전 에이전트 유지) ③ Claude 분류
   ↓ 에이전트 선택
[전문 에이전트] 시스템 프롬프트 + 허용 도구 + 대화 히스토리 + (학교 RAG 스코프) + (사용자 기억)
   ↓ Claude Opus 4.8 — tool_use
[도구] search_source(교재 RAG) / search_problem(저장 문제) / generate_problem / evaluate_answer / assess_level
   ↓
[랩업 스트리밍] 같은 프로필 보이스로 SSE 응답 → agent_messages 저장 → (교감 에이전트) 기억 추출
```

- 멀티에이전트는 별도 SDK가 아니라 패턴: 에이전트 = `AgentProfile`(시스템 프롬프트 + 도구 서브셋 + 행동 플래그).
- 생성·판정·OCR은 전부 Claude(`claude-opus-4-8`), 임베딩만 Gemini(768d pgvector 호환).

## 3. 에이전트 로스터 (9개)

| 에이전트 | 라벨 | 역할 | 핵심 루프 | 도구 |
|---|---|---|---|---|
| `recite` 🆕 | 암기 | **본문 외웠는지 확인** | 본문 로드 → 빈칸/해석→영작/문장잇기 테스트 → 원문 대조 채점(관사·시제까지) → 진행률 → 오답 재테스트 | search_source |
| `reading` 🆕 | 독해 | **독해력 점검** | 본문 로드 → 주제·요지·세부·추론 질문 → 학생 답 → 근거 문장 짚으며 채점 → 다음 질문 | search_source, search_problem |
| `grammar` | 문법 | 문법 개념 + 훈련 | 개념 설명 → 저장 문제 1개 출제(정답 숨김) → 채점+이유 → 유사 문제 | search_source, search_problem |
| `vocab` | 어휘 | 어휘 학습 + 훈련 | 뜻·용례 설명 → 어휘 문제 출제 → 채점+이유 → 유사 문제 | search_source, search_problem |
| `socratic` | 산파술 | 스스로 풀게 유도 | 유도 질문 → 학생 답 → **학생이 이유를 설명**하게 → 다음 단계 힌트 | search_source, search_problem |
| `problem_finder` | 문제 | 문제 검색·출제 | 검색/생성 → 풀게 함(정답은 카드 토글) → 채점+해설 → 유사 문제 | search_problem, generate_problem |
| `companion` | 잡담 | **교감** — 수다·농담 | 기억 회상("저번에 모카 얘기했죠") + 가벼운 대화 | (기억) |
| `emotion` | 감정 | **교감** — 정서 케어 | 공감 → 검증 → 작은 제안, 위기 시 전문기관 안내 | (기억) |
| `general` | — | 그 외 전부 | 답변 + 가능하면 학습 루프 연결 | 전체 |

## 4. 라우팅 규칙 (우선순위순)

1. **키워드 룰** (LLM 0회): emotion → recite → reading → problem_finder → grammar → vocab → socratic → companion
   - recite: `본문 외/암기 확인/외웠는지/빈칸 테스트…`
   - reading: `독해/주제 파악/요지/내용 일치/추론 문제…`
2. **Sticky (튜터링 연속성)**: 키워드 미적중 + 직전 어시스턴트 턴의 에이전트가 있으면 그대로 유지 — "③번", "왜?", 영작 답안 등 답변 턴이 루프를 끊지 않음.
3. **Claude 분류** (영어 과목 한정 폴백): 9개 enum 구조화 출력.

## 5. 컨텍스트 조립 (모든 에이전트 공통)

- **대화 히스토리**: 직전 12턴(턴당 3,000자 컷)을 Claude messages로 전달. 출제→답변→채점 루프의 전제.
- **학교 RAG**: 학교 선택 시 그 학교 소스로 search_source 범위 제한 + 시스템 프롬프트에 학교 컨텍스트.
- **사용자 기억**: companion/emotion은 `agent_memories` 최근 30개 주입, 턴 종료 후 기억 추출.
- **audience 게이팅**: student에게는 정답 노출 도구 제거(기존 유지). recite/reading의 채점은 학생 답 이후의 피드백이므로 허용.

## 6. 학습 루프 공통 규약 (시스템 프롬프트에 명시)

1. 문제·테스트를 낼 때 **정답·해설을 같은 턴에 쓰지 않는다**. "풀어보세요/말해보세요"로 답을 기다린다.
2. 학생 답이 오면 히스토리의 직전 문제 기준으로 **정오 판정 + 왜 그런지 근거 설명**. 가능하면 "왜 그렇게 골랐어요?"로 학생의 이유를 끌어낸다.
3. 한 사이클이 끝나면 **search_problem으로 비슷한 문제 1개**를 이어서 제시(없으면 같은 포인트로 직접 출제).
4. 진행 중인 테스트가 있으면 새 주제로 끊지 않는다 (sticky 라우팅이 보장).

## 7. 데이터

- 교재: `sources`(메타) + `source_chunks`(800자 청크 + 768d 임베딩) — 서울예고 14개 소스/392청크.
- 문제: `problems` — 316개 인제스트(정답지 페어 52, AI 판독 264 `검수 필요`), 임베딩 완료.
- 기억: `agent_memories` (student_id 단위).

## 8. 비범위 (추후)

- 학생별 오답노트/진도 대시보드 (`student_attempts` 활용 확장)
- recite 진행 상태의 구조화 저장(현재는 대화 히스토리로 추적)
- 음성 입력(낭독 암기 확인)
